# 就活企業分析 自動化ツール（n8n × Claude API × Tavily）

就職活動で数十社の企業を比較する非効率さを解決するために、自分の志向に合う企業をAIが提案・分析し、スプレッドシートに自動でまとめる個人開発ツールです。n8n上で構築しています。

> 個人の就活課題を起点に開発し、「使いながら課題を見つけて改善し続ける」ことを重視して作りました。

---

## 開発の背景（解決したかった課題）

就活で企業を比較する中で、自分一人で調べられる企業数には限界があると感じました。1社ずつ手で調べると時間がかかりすぎて、十分な数を比較できません。そこで、もともと興味のあったAI・API連携の技術を使い、「自分の志向に合う企業をAIが提案し、その企業を自動で分析してくれる」仕組みを作ることにしました。

---

## 何ができるか

1. 自分のガクチカ・志向を入力すると、相性の良い企業を5社提案する
2. 各企業について Web検索で最新情報を集める
3. 集めた情報をAIが分析し、「評判・年収・労働環境・備考」を要約する
4. 結果を Google スプレッドシートに自動で追記する

---

## 構成（ワークフロー）

```
手動実行
  ↓
① 企業提案 AI（Claude）  ── 志向に合う企業5社を構造化データで提案
  ↓
② Split Out ＋ Loop Over Items  ── 5社を1社ずつ処理
  ↓
③ Web検索（Tavily）  ── 各社の最新情報を取得
  ↓
④ 企業分析 AI（Claude）  ── 評判/年収/労働環境/備考に要約
  ↓
⑤ Google Sheets  ── 結果を自動追記
```

## 使用技術

| 技術 | 役割 |
|---|---|
| n8n | ワークフロー自動化（全体のオーケストレーション） |
| Claude API | 企業提案（Chat Model）／企業分析（Chat Model） |
| Tavily | Web検索（企業の最新情報取得・URLの実在性担保） |
| Google Sheets | 分析結果の蓄積 |
| Structured Output Parser | AI出力を構造化データとして安全に受け取る |

---

## 設計で工夫した点

技術の派手さより、「なぜそう作ったか」を大事にしました。

### 1. AI出力を構造化して壊れにくくした
当初はAIにJSON形式で出力させ、正規表現で無理やり整形していましたが、出力が少し崩れるだけで処理が壊れました。そこで **Structured Output Parser** に置き換え、スキーマ（データの型）を宣言的に定義することで、脆い文字列処理を排除しました。

### 2. AIにURLを生成させない（ハルシネーション対策）
初期はAIに企業の採用URLを生成させていましたが、存在しないURLを返すことがありました。これを **Tavily のWeb検索で実在URLと最新情報を取得する** 方式に変更し、情報の正確性と鮮度を確保しました。

### 3. エラーに強い設計
- 外部API呼び出しに **Retry On Fail**（自動再試行）を設定
- **Loop Over Items** で1社ずつ処理し、1社の失敗が全体を止めないようにした
- **Error Workflow** を用意し、失敗を検知できるようにした
- Google Sheets を **Append or Update** にして、再実行時の重複を防止

## セットアップ方法

このワークフローを動かすには、各自で以下の認証情報（クレデンシャル）を設定する必要があります。**本リポジトリのJSONには、APIキー・スプレッドシートID・個人情報は含まれていません**（すべてプレースホルダに置換済み）。

1. n8n に `company-research-automation.json` をインポート
2. 以下のクレデンシャルを各自の環境で設定
   - **Anthropic API**（Claude APIキー）
   - **Tavily API**（[app.tavily.com](https://app.tavily.com) で無料取得、月1,000クレジット）
   - **Google Sheets OAuth2**
3. Google Sheets ノードの `documentId`（`YOUR_GOOGLE_SHEET_ID`）を自分のスプレッドシートIDに差し替え
4. スプレッドシートの1行目に列見出しを用意：
   `kaisha / riyuu / hyoka / nenshu / kyujitsu / biko / url / score / score_riyuu / ir_nenshu / ir_summary / updated_at`
5. 企業提案プロンプト（Basic LLM Chain）内のプレースホルダに、自分のガクチカ・志向を記入

## 既知の制約

- 現時点では手動実行（Manual Trigger）で動作します。定期自動実行やUIは未対応です。
- スコアリング・IR資料分析は開発中で、本バージョンには未実装です。
- 未上場企業は公式開示情報が存在しないため、Web検索ベースの推定情報になります。
