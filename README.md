# 企業分析自動化ツール（company-research-automation）

就職活動の企業研究を自動化する、n8n × Claude API の個人開発ツールです。
「志向に合う企業を提案 → Web検索で情報収集 → AIで分析 → スプレッドシートに一覧化」までを、実行ボタン1つで完結させます。

## 開発の背景

就職活動では、1社ごとに事業内容・評判・年収・労働環境を調べてまとめる作業に多くの時間がかかります。この繰り返し作業を自動化すれば、比較検討そのものに時間を使えると考え、プログラミング未経験の状態から独学でこのツールを作りました。

## できること

- 自分のガクチカ・志向を渡すと、相性の良い企業を5社提案する（過去の提案と重複しない）
- 各企業をWeb検索し、評判・強み・年収・労働環境などをAIが要約する
- 自分の志向（上流工程／AI活用／育成／若手裁量）への適合度を10点満点で採点する
- 結果をGoogleスプレッドシートに自動で追記・更新する

## 処理の流れ

```
手動実行
  ↓
① 企業提案（Claude Sonnet）… 条件に合う企業を5社、企業名＋推薦理由で出力
  ↓
② Split Out … 5社を1件ずつに分解
  ↓
③ Loop Over Items … 1社ずつ順に処理
  ↓
④ Web検索（Tavily）… 「企業名 採用 年収 評判」で情報収集
  ↓
⑤ 企業分析（Claude Opus）… 評判・年収・労働環境・特記事項＋適合度スコアを生成
  ↓
⑥ Googleスプレッドシートに追記・更新
  ↺ ③に戻り次の企業へ
```

## 使用技術

| 分類 | 使用したもの |
| --- | --- |
| 自動化基盤 | n8n |
| LLM | Anthropic Claude API（提案：Sonnet／分析：Opus） |
| Web検索 | Tavily |
| 出力先 | Google Sheets API |
| 出力の構造化 | Structured Output Parser |

## セットアップ

1. `企業分析_新_開発中.json` を n8n にインポートする
2. 認証情報を各自のものに設定する（Anthropic API／Tavily／Google Sheets）
3. 「Append or update row in sheet」ノードの **Document** を、自分のGoogleスプレッドシートに設定する（公開用にIDは伏せてあります）
4. 「①企業提案」ノードのプロンプト内のガクチカ・志向を、自分の内容に書き換える
5. 手動実行する

## 設計上の判断

このツールで工夫したのは、機能を増やすことより「うまくいかない原因を見極めて作り直す」ことです。

**1. 出力の受け取りを正規表現からStructured Output Parserへ**
当初はAIの回答から正規表現で項目を抜き出していましたが、回答の書き方が少し変わるだけで抽出が崩れ、不安定でした。決まった形式（スキーマ）で受け取るStructured Output Parserに置き換え、後続の処理が安定して動くようにしました。

**2. URLの取得をAI生成からTavily検索へ**
最初はAIに企業の情報源URLを生成させていましたが、存在しないURLを返すことが多く、情報の裏取りができませんでした。実際のWeb検索を行うTavilyに切り替え、実在する情報に基づいて分析するようにしました。

**3. EDINET連携（IR取得機能）の廃止**
上場企業の有価証券報告書をEDINET APIから取得する機能も組み込んでいましたが、開発の中で3つの制約が分かりました。有価証券報告書は年に1回しか提出されず特定企業の抽出に大量の書類取得が必要なこと、企業ごとに決算期が違い取得範囲を固定すると見つからないこと、非上場企業はそもそも対象にできないこと、の3点です。
このツールで本当に必要だったのは平均年収などの数値そのものより事業内容や強みの「説明」であり、それはTavily検索で取得できていたため、取得コストの高いEDINET連携は目的に見合わないと判断し、機能ごと削除してTavily検索に一本化しました。

## 工夫した実装上のポイント

- **Retry On Fail**：API呼び出しノードに再試行（5秒間隔）を設定し、一時的なエラーで全体が止まらないようにした
- **Loop Over Items**：企業を1社ずつ処理し、レート制限や部分的な失敗に強い構成にした
- **appendOrUpdate**：同じ企業を再実行したとき、行を重複させず更新する設定にした
