# 就活企業分析 自動化ツール（n8n × Claude API × Tavily × EDINET）

就職活動で数十社の企業を比較する非効率さを解決するために、自分の志向に合う企業をAIが提案・分析・採点し、有価証券報告書の一次情報まで自動でまとめる個人開発ツールです。n8n上で構築しています。

> 個人の就活課題を起点に開発し、「使いながら課題を見つけて改善し続ける」ことを重視して作りました。単発の情報収集ツールから、意思決定を支援するシステムへと段階的に育てています。

---

## 開発の背景（解決したかった課題）

就活で企業を比較する中で、自分一人で調べられる企業数には限界があると感じました。1社ずつ手で調べると時間がかかりすぎて、十分な数を比較できません。そこで、もともと興味のあったAI・API連携の技術を使い、「自分の志向に合う企業をAIが提案し、その企業を自動で分析・採点してくれる」仕組みを作ることにしました。

---

## 何ができるか

1. 自分のガクチカ・志向を入力すると、相性の良い企業を5社（証券コード付き）で提案する
2. 各企業について Web検索で最新情報を集める
3. 集めた情報をAIが分析し、「評判・年収・労働環境・備考」を要約する
4. 自分の志向との適合度を10点満点でスコアリングする
5. 上場企業は有価証券報告書（EDINET API）から平均年収などの一次情報を取得・要約する
6. 結果を Google スプレッドシートに自動でまとめる（スコア順にソート可能）

## 構成

### メインワークフロー
```
手動実行
  ↓
① 企業提案 AI（Claude）  ── 志向に合う企業5社＋証券コードを構造化データで提案
  ↓
② Split Out ＋ Loop Over Items  ── 5社を1社ずつ処理
  ↓
③ Web検索（Tavily）  ── 各社の最新情報を取得
  ↓
④ 企業分析＋スコアリング AI（Claude）  ── 評判/年収/労働環境/備考＋適合度スコア
  ↓
⑤ Google Sheets  ── 結果を自動記録
```

### IR分析サブワークフロー（EDINET連携）
```
証券コードを受け取る（Execute Workflow Trigger）
  ↓
書類一覧API → 目的企業の有価証券報告書を特定（docID）
  ↓
書類取得API（CSV形式）→ ZIP解凍 → テキスト抽出
  ↓
「従業員の状況」から平均年収・勤続年数・年齢を抽出
  ↓
AIが事業内容・新規事業・研究開発とあわせて要約
```

※ 構成図（ワークフロー全体のスクリーンショット）をここに貼ると分かりやすくなります。

## 使用技術

| 技術 | 役割 |
|---|---|
| n8n | ワークフロー自動化（全体のオーケストレーション、サブワークフロー分割） |
| Claude API | 企業提案／企業分析／スコアリング／IR要約 |
| Tavily | Web検索（企業の最新情報取得・URLの実在性担保） |
| EDINET API（金融庁） | 有価証券報告書の取得（平均年収などの一次情報） |
| Google Sheets | 分析結果の蓄積 |
| Structured Output Parser | AI出力を構造化データとして安全に受け取る |

## 設計で工夫した点

技術の派手さより、「なぜそう作ったか」を大事にしました。

### 1. AI出力を構造化して壊れにくくした
当初はAIにJSON形式で出力させ、正規表現で無理やり整形していましたが、出力が少し崩れるだけで処理が壊れました。そこで **Structured Output Parser** に置き換え、スキーマ（データの型）を宣言的に定義することで、脆い文字列処理を排除しました。

### 2. AIにURLを生成させない（ハルシネーション対策）
初期はAIに企業の採用URLを生成させていましたが、存在しないURLを返すことがありました。これを **Tavily のWeb検索で実在URLと最新情報を取得する** 方式に変更し、情報の正確性と鮮度を確保しました。

### 3. 情報の種類ごとに抽出方法を変えた（IR分析）
有価証券報告書は数百ページあり、Claude APIのPDFページ上限（100ページ）に抵触します。そこで **PDFではなくCSV形式（EDINET type=5）で取得** し、平均年収などの数値はXBRLの要素IDでピンポイント抽出、事業内容などの長文はセクション単位で抽出してAIに要約させる、という **データの性質に応じた抽出設計** にしました。

### 4. ワークフローを分割した（保守性）
IR分析は処理が複雑なため、メインワークフローに混ぜず **サブワークフローとして分離** し、Execute Workflowで呼び出す構成にしました。これにより本体の見通しを保ち、IR機能単体でのテスト・改修をしやすくしました。

### 5. エラーに強い設計
- 外部API呼び出しに **Retry On Fail**（自動再試行）を設定
- **Loop Over Items** で1社ずつ処理し、1社の失敗が全体を止めないようにした
- **Error Workflow** を用意し、失敗を検知できるようにした
- Google Sheets を **Append or Update** にして、再実行時の重複を防止

## 進化の物語

このツールは一度で完成したものではなく、使いながら改善を重ねました。

```
単発の1社ツール
  → 複数社を一括処理できるように拡張
  → Web検索を追加し、情報の鮮度と実在性を確保
  → 出力を構造化し、壊れにくい設計に作り替え
  → 志向との適合度スコアで「意思決定支援」に踏み込む
  → 有価証券報告書の一次情報分析を追加（現在ここ）
```

## 今後の拡張予定

- **IR分析の全社対応**：現在、有価証券報告書の取得は書類提出日を指定する実装のため、提出日が既知の企業（実証済み：トヨタ自動車＝平均年収約1,006万円を自動抽出）で動作します。全企業に対応するには、提出集中期間（6月など）を走査して企業を特定する期間ループの実装が必要で、これを次の改善項目としています。
- 定期自動実行（Schedule Trigger）とインターン締切の監視
- 分析品質の定量評価（Evaluations）

## セットアップ方法

このワークフローを動かすには、各自で以下の認証情報（クレデンシャル）を設定する必要があります。**本リポジトリのJSONには、APIキー・スプレッドシートID・個人情報は含まれていません**（すべてプレースホルダに置換済み）。

1. n8n に `company-research-automation.json`（メイン）と `edinet-ir-analysis.json`（IR分析サブ）をインポート
2. 以下のクレデンシャル／キーを各自の環境で設定
   - **Anthropic API**（Claude APIキー）
   - **Tavily API**（app.tavily.com で無料取得、月1,000クレジット）
   - **Google Sheets OAuth2**
   - **EDINET API**（金融庁EDINETで無料取得、`YOUR_EDINET_API_KEY` を置換）
3. Google Sheets ノードの `documentId`（`YOUR_GOOGLE_SHEET_ID`）を自分のスプレッドシートIDに差し替え
4. スプレッドシートの1行目に列見出しを用意：
   `kaisha / riyuu / hyoka / nenshu / kyujitsu / biko / url / score / score_riyuu / ir_nenshu / ir_summary / updated_at`
5. 企業提案プロンプト（Basic LLM Chain）内のプレースホルダに、自分のガクチカ・志向を記入
6. メインWFの Execute Workflow ノードで、IR分析サブワークフローを指定

## 既知の制約

- 現時点では手動実行（Manual Trigger）で動作します。定期自動実行やUIは未対応です。
- IR分析は書類提出日指定の実装のため、全企業での自動取得には期間ループの実装が必要です（今後の拡張参照）。
- 未上場企業は有価証券報告書が存在しないため、Web検索ベースの情報になります。
